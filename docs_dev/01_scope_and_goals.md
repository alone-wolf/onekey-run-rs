# 项目范围与目标

## 1. 产品定位

`onekey-run-rs` 是一个基于 Rust 实现的轻量级 CLI 工具，用于在当前工作目录读取 `onekey-tasks.yaml`，并按照配置启动、监控和停止一组本地服务进程。

它更接近“无容器的本地进程编排器”，而不是通用进程守护系统或分布式调度平台。

## 2. 目标用户

- 需要在本地快速拉起多服务开发环境的开发者
- 需要通过单个命令启动一组依赖进程的小型项目
- 希望用统一 YAML 取代手写 shell 脚本的团队

## 3. v1 目标

- 从当前目录自动发现并读取 `onekey-tasks.yaml`
- 根据依赖关系按顺序启动服务
- 提供基于 PID 的基础进程状态管理
- 聚合并输出多服务日志
- 提供 `init`、`check`、`up`、`down` 四个首版命令
- 在收到退出信号或执行 `down` 时优雅停止全部子进程
- 对配置错误和运行失败给出清晰错误信息

## 4. v1 非目标

以下内容明确不纳入首版：

- 远程节点调度
- 集群管理
- Web UI
- 热重载配置
- 持久化运行状态数据库
- 与容器编排系统的深度集成
- 进程健康检查和业务级 ready 判定

## 5. 成功标准

项目进入可用状态的最低标准：

- 用户可通过 `onekey-run up` 启动一个多服务项目
- 用户可通过 `onekey-run down` 可靠停止该项目已启动的服务
- 依赖顺序和 PID 存活语义符合文档定义
- `Ctrl-C`、`SIGTERM` 或 `down` 可触发可靠清理
- 常见配置错误可在启动前被 `check` 命令发现
- 集成测试可覆盖核心状态流转

## 6. 设计边界

规划期已确认以下边界：

- 首版命令集为 `init`、`check`、`up`、`down`
- 首版不做配置 `version` 字段
- 首版不做健康检查和业务级 ready，仅以进程是否仍然存活为基础状态语义
- 平台目标为同时支持 Unix-like 和 Windows，但是否一次性交付需要单独评估

## 7. 待确认问题

- Windows 首版是否和 Unix-like 完全同等能力，还是采用分阶段支持
- `up` 是否默认支持仅启动部分服务及其依赖
- 服务是否允许配置 prestart / poststop hooks
